ReleaseOps Arena

Oversight for CI/CD: training a “release safety” agent via RL, JSON observations, tool actions, and terminal rewards.

Scenario State (What the model sees)

Structured JSON: release phase (e.g. proposal → review → cutover), proposals to approve or block, CI run ids and job status, policy rules, and budgets. The agent calls tools (inspect_pr_diff, ask_worker) before deciding to approve, block, or hold.

Episode & Reward Architecture

flowchart TD
    A["Start Node
Scenario & Budgets"] --> B["Observation JSON"]
    B --> C{"Agent Action"}

    C --> C1["Inspect Evidence"]
    C --> C2["Decide (Approve/Block)"]
    C --> C3["Hold Release"]

    C1 --> D1["State update: Consume budget"]
    C2 --> D2["State update: Proposal resolved"]
    C3 --> D3["State update: Advance time"]

    D1 -.-> S1[Shaping +0.2 violation found]
    D2 -.-> S3[Shaping -0.15 false block]

    D1 --> E{"Terminal?"}
    D2 --> E
    D3 --> E

    E -- No --> B
    E -- Yes --> T{"Outcome"}

    T --> T1[Safe ship] --> R1[Reward +1.0]
    T --> T2[Unsafe ship] --> R2[Reward -1.0]
    T --> T3[Missed deadline] --> R3[Reward -0.6]
Main Transition step to step Shaping Reward applied instantly
  • Before RL: We run CI/CD with no one in the middle. The untrained model acts like a naive script, pushing failures to production.
  • After RL: The oversight agent inspects diffs and logs. It acts as a required logic gate, securing the release path.
RL/GRPO Architecture: Shaping rewards train the agent to gather evidence step-by-step. The terminal reward dictates the final success (safe ship vs. unsafe ship).
